    • 1. Inline CSS

    Inline CSS is a method of styling HTML elements directly within the HTML tag itself. It uses the *style attribute*.

    How It Works: You apply CSS rules to a specific HTML element. The styles are written directly inside the opening tag of the element. This method is used for styling a single element only.

    Example:

    HTML

    <p style="color: blue; font-size: 16px;">This paragraph is styled using inline CSS.</p>

    In this example, the p element has a style attribute that sets the text color to *blue* and the font size to *16 pixels*.

    Advantages:

    • Easy and quick for a single, unique style.
    • Has a high precedence, meaning it will override styles from internal or external stylesheets.

    Disadvantages:

    • Makes HTML files cluttered and difficult to read.
    • Not reusable; you have to write the same code for every element you want to style.
    • Hard to manage and maintain for larger projects.

    2. Internal CSS

    Internal CSS, also known as *embedded CSS, is used to style a single HTML page. The styles are placed within a **<style> tag* in the *<head>* section of the HTML document.

    How It Works: The *<style>* tag contains all the CSS rules for the entire page. You use selectors (like h1, p, or classes/IDs) to target and style multiple elements.

    Example:

    HTML structure (Code from the <head> section is at the top of this document):

    <head> ... <style> body { background-color: lightgray; } h1 { color: green; text-align: center; } </style> </head> <body> <h1>This is a heading styled with internal CSS.</h1> <p>This page has a light gray background.</p> </body>

    In this example, the h1 element is targeted and styled to have a *green color* and be *centered. The body element has a **light gray background*.

    Advantages:

    • Useful for styling a single page that has a unique style.
    • All styles for the page are in one place, making it easy to find and edit.

    Disadvantages:

    • Styles cannot be reused across multiple pages.
    • Still makes the HTML file larger and can be difficult to manage for very long stylesheets.

    3. External CSS

    External CSS is the most common and recommended method for styling websites. The styles are written in a *separate file with a .css extension* and then *linked* to the HTML document.

    How It Works: You create a separate file (e.g., styles.css) containing all your CSS rules. You link this file to your HTML document using the *<link> tag* in the *<head>* section.

    Example:

    HTML File (index.html):

    <head> ... <link rel="stylesheet" href="styles.css"> </head> <body> <h2>This is an external CSS heading.</h2> <p>This text is styled from the linked stylesheet.</p> </body>

    CSS File (styles.css):

    h2 { color: purple; font-family: Arial, sans-serif; } p { line-height: 1.5; color: #333; }

    In this example, the HTML file links to styles.css. The styles defined in that CSS file are applied to the HTML elements.

    Advantages:

    • Separates content (HTML) from presentation (CSS), making the code cleaner and easier to manage.
    • Highly reusable: A single .css file can be linked to multiple HTML pages, ensuring a consistent look and feel across the entire website.
    • Makes site-wide changes easy; you only have to edit one file to update styles on all pages.
    • Reduces file size and improves loading times as the CSS file is cached by the browser.

    Disadvantages:

    • Requires an extra file download for the browser.
    • The page won't be styled until the external CSS file is loaded.
